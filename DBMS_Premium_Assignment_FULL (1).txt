  Q1. Evaluate ANSI-SPARC Architecture
  --------------------------------------------------
  The ANSI-SPARC architecture is a three-level
  database architecture:

  1. External Level: - Provides different views to
  users - Ensures data security and customization

  2. Conceptual Level: - Represents overall logical
  structure - Defines entities, relationships,
  constraints

  3. Internal Level: - Describes physical storage of
  data - Includes indexing and file organization

  Data Independence: - Logical Data Independence:
  Change in conceptual schema does not affect user
  views - Physical Data Independence: Change in
  storage does not affect logical schema

  Conclusion: This architecture ensures flexibility,
  scalability, and easy maintenance.

Q2. Analyze Mapping Cardinalities

Mapping cardinality defines relationships between entities:

1.  One-to-One (1:1) Example: Person – Passport Impact: Can merge tables

2.  One-to-Many (1:M) Example: Customer – Orders Impact: Foreign key in
    many-side table

3.  Many-to-One (M:1) Reverse of 1:M

4.  Many-to-Many (M:N) Example: Students – Courses Impact: Requires
    junction table

Conclusion: Cardinality affects table structure, keys, and database
efficiency.

Step 1: Create Table

CREATE TABLE Hotel ( Hotel_ID VARCHAR2(5) PRIMARY KEY, Name VARCHAR2(50)
NOT NULL, Location VARCHAR2(50) NOT NULL, Rating NUMBER(1) CHECK (Rating
BETWEEN 1 AND 5), Contact_No VARCHAR2(10) UNIQUE, Email VARCHAR2(50)
UNIQUE, Total_Rooms NUMBER(5) CHECK (Total_Rooms > 0), Manager_Name
VARCHAR2(50) NOT NULL );
